goal is to build Lean finetuning data from mathlib by generating standalone recreation files, one theorem per file, that compile on their own and stay as close as possible to the original mathlib proof text.

The core rules you’ve been aiming for are:

each file should include the theorem plus the minimum relevant context needed to compile

keep import Mathlib at the top rather than importing a deep subpath in the final recreation format

preserve theorem proofs essentially unchanged, allowing only necessary prefix additions and similarly minimal rewrites

avoid open in recreation files, so names are explicit and stable

preserve the folder structure from Mathlib/... under LeanWorkspace/.../Recreations/...

generate these files programmatically and test them automatically with Lean

And philosophically, the dataset should “pretend we know mathlib up to that theorem, but not the theorem itself.”
